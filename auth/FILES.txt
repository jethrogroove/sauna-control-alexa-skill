12. lib/supabase.js (200 lines)
    - Supabase client initialization
    - Public client for signup/signin
    - Service role client for server operations
    - Helper functions:
      - signUpUser(email, password)
      - signInUser(email, password)
      - getUserById(userId)

13. lib/crypto.js (150 lines)
    - AES-256-GCM encryption/decryption
    - Derives key from ENCRYPTION_KEY env var
    - Functions:
      - encryptCredential(plaintext) -> {encrypted, iv, authTag}
      - decryptCredential(encrypted, iv, authTag) -> plaintext
      - encryptSaunaCredentials(credentials) -> encrypted object
      - decryptSaunaCredentials(record) -> plaintext object
    - 128-bit random IV per encryption
    - Auth tag prevents tampering

14. lib/tokens.js (220 lines)
    - OAuth 2.0 token management
    - Authorization codes (5 min expiry, single-use)
    - Access tokens (JWT, 1 hour expiry)
    - Refresh tokens (opaque, 30 day expiry)
    - Functions:
      - createAuthorizationCode(userId, clientId, redirectUri)
      - verifyAuthorizationCode(code, clientId, redirectUri)
      - createAccessToken(userId, expiresInSeconds)
      - verifyAccessToken(token)
      - createRefreshToken(userId, clientId, expiresInSeconds)
      - verifyRefreshToken(token, clientId)
      - revokeRefreshToken(token)

DATABASE (1 file)
================================================================================

15. supabase-schema.sql (100+ lines)
    - PostgreSQL schema for Supabase
    
    Tables created:
    
    1. sauna_credentials
       - Encrypted email and password for sauna providers
       - IV and auth tag for each credential
       - User and provider foreign keys
       - Timestamps for audit trail
       - Unique constraint on (user_id, provider)
    
    2. oauth_codes
       - Single-use authorization codes
       - Links to user, client, redirect URI
       - Expiry timestamp (5 minutes)
       - Used flag to prevent reuse
    
    3. oauth_refresh_tokens
       - Long-lived refresh tokens
       - Links to user and client
       - Expiry timestamp (30 days)
       - Revoked flag for explicit revocation
    
    Row-Level Security (RLS):
    - sauna_credentials: User can access own records
    - oauth_codes: Service role only
    - oauth_refresh_tokens: Service role only
    
    Indexes created:
    - Primary keys on all tables
    - Foreign key indexes for joins
    - Unique indexes on codes/tokens
    - Expiry indexes for cleanup queries
