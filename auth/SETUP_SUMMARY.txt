QUICK SETUP (15 MINUTES):

1. CREATE SUPABASE PROJECT
   - Go to supabase.com and create new project
   - Note: SUPABASE_URL, SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY
   - Run supabase-schema.sql in Supabase SQL Editor

2. GENERATE SECRETS
   node -e "console.log('ENCRYPTION_KEY=' + require('crypto').randomBytes(32).toString('base64'))"
   node -e "console.log('JWT_SECRET=' + require('crypto').randomBytes(32).toString('base64'))"
   node -e "console.log('ALEXA_CLIENT_ID=' + require('crypto').randomBytes(16).toString('hex'))"
   node -e "console.log('ALEXA_CLIENT_SECRET=' + require('crypto').randomBytes(32).toString('hex'))"

3. CREATE .env.local
   SUPABASE_URL=<your-url>
   SUPABASE_ANON_KEY=<your-anon-key>
   SUPABASE_SERVICE_ROLE_KEY=<your-service-role-key>
   ENCRYPTION_KEY=<generated-key>
   ALEXA_CLIENT_ID=<generated-id>
   ALEXA_CLIENT_SECRET=<generated-secret>
   JWT_SECRET=<generated-secret>

4. RUN LOCALLY
   npm install
   npm run dev
   Visit: http://localhost:3000/authorize?client_id=test&redirect_uri=http://localhost:3000&response_type=code

5. DEPLOY TO VERCEL
   npm i -g vercel
   vercel login
   vercel deploy

6. CONFIGURE ALEXA SKILL
   - Go to Alexa Developer Console > Account Linking
   - Authorization URL: https://your-vercel-url.vercel.app/authorize
   - Access Token URL: https://your-vercel-url.vercel.app/api/oauth/token
   - Client ID: (from ALEXA_CLIENT_ID)
   - Client Secret: (from ALEXA_CLIENT_SECRET)

API ENDPOINTS:

POST /api/auth/signin
  Request:  { email, password }
  Response: { userId, email }

POST /api/auth/signup
  Request:  { email, password }
  Response: { userId, email }

GET /authorize?client_id=X&redirect_uri=Y&response_type=code&state=Z
  Returns: OAuth authorization page

POST /api/oauth/authorize
  Request:  { userId, clientId, redirectUri, provider, saunaEmail, saunaPassword }
  Response: { authCode }

POST /api/oauth/token
  Request:  { grant_type, code, redirect_uri, client_id, client_secret }
          OR { grant_type, refresh_token, client_id, client_secret }
  Response: { access_token, token_type, expires_in, refresh_token }

GET /api/credentials (Bearer <token>)
  Response: { provider, email, password }

DATABASE SCHEMA:

sauna_credentials
  - id (UUID)
  - user_id (references auth.users)
  - provider (varchar)
  - encrypted_email, iv_email, auth_tag_email
  - encrypted_password, iv_password, auth_tag_password
  - created_at, updated_at

oauth_codes
  - id (UUID)
  - code (varchar, unique)
  - user_id (references auth.users)
  - client_id, redirect_uri
  - expires_at
  - used (boolean)
  - created_at

oauth_refresh_tokens
  - id (UUID)
  - token (varchar, unique)
  - user_id (references auth.users)
  - client_id
  - expires_at
  - revoked (boolean)
  - created_at

SECURITY FEATURES:

✓ Encryption at Rest
  - Sauna credentials encrypted with AES-256-GCM
  - Each credential has unique IV and auth tag
  - Encryption key never exposed to client

✓ Token Security
  - Access tokens are short-lived JWTs (1 hour)
  - Refresh tokens are opaque and can be revoked
  - Auth codes single-use and expire after 5 minutes
  - All tokens validated server-side

✓ Database Security
  - Row-Level Security on all tables
  - Service Role key kept secret
  - Indexes on all foreign keys and queries

✓ Transport Security
  - HTTPS required in production (automatic on Vercel)
  - Secure cookies with HttpOnly, SameSite flags
  - CSRF protection via state parameter
